Report Manager for Dot Net

Basics

The dot net version of Report Manager is a translation from Delphi to C# of the reporting engine. The port to dot net is still uncomplete and requires still some work, but you can test it easily. A library is provided so you can link easily your reports with your application.

How it works now (while testing the port)

A single additional file is installed with the designer package for windows, the file is printreport.exe, this is a dot net program. In Linux Mono 1.1.13 runtime must be installed, in Microsoft Windows Dot Net runtime Framework 1.1 must be installed.

When you select the dot net driver on database connections configuration, the designer, will call printreport.exe to show you data results in data configuration dialog, and to preview/print the report.

The internal procedure is save the report in a temp file, as XML (see page setup, prefered save format), then call printreport.exe with command line options (-preview for example). That is when you preview or print, the printreport.exe is used instead of the designer itself. You can also test printreport.exe independently, the only requirement is you save the report as xml (select prefered save format in page setup).

For example, this instruction will generate a pdf file from a report:

printreport.exe -pdf pdfoutputfilename.pdf mysavedreportasxml.rep

Parts already working (Windows)

  • Print preview and printout
  • PDF output (not truetype support)
  • Label, expression, image, drawing, barcode (partial)
  • Data providers: Odbc, OleDB, SQLServer, Firebird, SQLite,Sybase,PostgreSQL,IBM DB2 (Linux),MySQL(Windows)

Missing parts

  • Output as excel, text
  • Some evaluator functions are still missing
  • Linux port, please use pdf output to printout on linux, the Mono printing components are not complete (Winforms still not implemented).
  • Some barcodes still missing
  • Other data providers may be included in the future: IBM DB2 (Windows), MySQL (Linux)

Different behaviour

  • You must use dot net syntax for parameters in querys, see documentation in dot net platform sdk.
  • In odbc provider you use ? for placing parameters, the parameters will be assigned by position, not by name, parameters by name are not supported by odbc dot net driver, so make sure to sort the parameters to match query position when you use odbc.

Using the library

To use the library provided (reportmanlib.dll), you should include a reference to reportmanlib.dll in your project.

This is sample code to execute a report, a portion of the sample provided testinglib.exe.

using Reportman;

Report rp=new Reportman.Report();
ReportReader rreader=new ReportReader(rp);
rreader.LoadFromFile(EFile.Text);
if (pdf.Checked)
{
rp.AsyncExecution=false;
PrintOutPDF printpdf=new PrintOutPDF();
printpdf.FileName=EPDFFile.Text;
printpdf.Compressed=false;
rp.BeginPrint(printpdf);
printpdf.Print(rp.metafile);
}
else
if (preview.Checked)
{
rp.AsyncExecution=true;
Reportman.PreviewControl prc=new PreviewControl();
prc.SetReport(rp);
Reportman.PreviewWinForms prw=new PreviewWinForms();
prw.PreviewReport(prc);
}
else
{
rp.AsyncExecution=false;
PrintOutWinForms printout=new PrintOutWinForms();
printout.Preview=systempreview.Checked;
printout.ShowPrintDialog=showprintdialog.Checked;
rp.BeginPrint(printout);
printout.Print(rp.metafile);
}
