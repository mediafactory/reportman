Report Manager for Dot Net

Basics

The dot net version of Report Manager is a translation from Delphi to C# of the reporting engine. The port to dot net is still uncomplete and requires still hard work.

How it works now (while testing the port)

A single additional file is installed with the designer package for windows, the file is printreport.exe, this is a dot net program.

When you select the dot net driver on database connections configuration, the designer will save the report in a temp file, calling printreport.exe, when you preview or print, the printreport.exe is used instead of the designer itself. You can also test printreport.exe independently, the only requirement is you save the report as xml (select prefered save format in page setup).

Parts already working (Windows)

  • Print preview and printout
  • Label, expression, image, drawing, barcode (partial)
  • Data providers: Odbc, OleDB, SQLServer, Firebird (firebird dotnet assembly required)

Missing parts

  • Output as pdf, excel, text
  • Most of the evaluator functions are still missing
  • Linux port, requires winforms on Linux or implementation of pdf output.
  • Some barcodes still missing
  • Other data providers may be included in the future (MySQL)

Different behaviour

  • You must use dot net syntax for parameters in querys, see documentation in dot net platform sdk.
  • In odbc provider you use ? for placing parameters, the parameters will be assigned by position, not by name, parameters by name are not supported by odbc dot net driver, so make sure to sort the parameters to match query position when you use odbc.
